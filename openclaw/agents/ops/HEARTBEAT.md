# 运营部心跳任务（每8小时执行，在所有其他Agent之前）

## 步骤1: 采集外部数据
- 通过 shell 调用百度统计API -> 官网PV/UV/来源/热门页面
- 通过 shell 调用 SendGrid API -> 邮件发送量/打开率/点击率/退订率
- 通过 shell 调用官网API GET /api/leads?stats=true -> 线索统计

## 步骤2: 采集内部数据
- 读取 /data/crm/leads/ -> 统计各级别线索数量和状态分布
- 读取 /data/crm/partners/ -> 统计合伙人线索数量和状态
- 读取各Agent最新报告的时间戳 -> 判断Agent是否正常运行
- 读取 /reports/sales/URGENT.md -> 检查是否有未处理的紧急事项

## 步骤3: 生成经营报表
写入 /reports/ops/latest.md：
```
# 微算经营仪表盘
更新时间: YYYY-MM-DD HH:00

## 核心指标
- 累计线索: X条
- 本周新增线索: X条
- A级线索: X条（列出名单）
- 热线索（有购买信号）: X条（列出名单）
- 已发送方案: X份
- 客户回复率: X%
- 微算-B成交: X单
- 累计收入: X元

## 销售漏斗
官网访问(X) -> 留资(X) -> 有效线索(X) -> 方案阶段(X) -> 成交(X)
转化率: X% -> X% -> X% -> X%

## 内容效果
- 本周发布: X篇博客 / X个知乎回答
- 博客总阅读: X
- 最受欢迎内容: "xxx"（阅读X）

## 各部门执行状态
- Scout: 最后执行 YYYY-MM-DD HH:MM | 本周搜索X家企业
- Content: 最后执行 YYYY-MM-DD HH:MM | 本周发布X篇内容
- Sales: 最后执行 YYYY-MM-DD HH:MM | 本周触达X家客户
- GM: 最后决策 YYYY-MM-DD HH:MM

## 异常告警
- [ ] Agent运行状态（如有Agent超过16小时未执行 -> 告警）
- [ ] A级线索处理（如有超过24小时未处理 -> 告警）
- [ ] 邮件发送（如失败率>20% -> 告警）
- [ ] 官网API可用性

## 距离目标
- 首单成交: 未完成/已完成
- 月度线索目标: X/30
- 月度触达目标: X/50
```

## 步骤4: 更新CRM统计
更新 /data/crm/stats.md 为最新统计数据。

## 步骤5: 异常告警
如检测到以下情况，通过企业微信API通知创始团队：
- 任何Agent连续2次心跳（16小时）未产生报告
- A级线索超过24小时未处理
- /reports/sales/URGENT.md 存在且未标记为"已通知"
- 邮件发送失败率超过20%

## 步骤6: 执行总经理额外命令
读取 /orders/ops/current.md，执行总经理可能下达的额外任务。
