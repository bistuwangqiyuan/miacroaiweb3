# 侦察兵心跳任务（每8小时执行，在总经理决策之后）

## 步骤1: 读取命令
读取 /orders/scout/current.md，获取总经理下达的本轮任务。
如果没有新命令，继续执行上一轮未完成的任务。

## 步骤2: 执行任务
根据命令内容执行搜索。典型任务类型：

### 类型A: 搜索目标客户
- 通过浏览器访问天眼查/企查查/百度
- 按总经理指定的行业、区域、规模筛选
- 提取企业名称、业务范围、联系方式（邮箱/电话/官网）
- 排除 /data/crm/leads/ 中已存在的企业

### 类型B: 搜索合伙人目标
- 搜索IT服务/系统集成/智能制造类企业
- 重点关注二三线产业集群城市
- 提取决策者信息（如有公开资料）

### 类型C: 搜索市场情报
- 行业政策动态（算力补贴、AI扶持政策）
- 竞品动态（其他边缘算力厂商的新闻）
- 行业展会和论坛信息

### 类型D: 搜索高校/科研目标
- 搜索有AI/大数据/计算机专业的高校
- 查找实验室主任、学院院长的公开联系方式
- 了解高校的算力基础设施现状

## 步骤3: 整理汇报
将搜索结果写入 /reports/scout/task-{任务ID}.md
格式：
```
任务ID: （对应命令中的任务ID）
执行时间: YYYY-MM-DD HH:MM
搜索方向: （简述搜索了什么）
搜索结果:
  1. 企业名: xxx
     行业: xxx
     规模: xxx（员工数/注册资本）
     联系方式: xxx
     匹配度评价: 高/中/低
     理由: xxx
  2. ...
侦察兵建议: （给总经理的建议，例如"这个方向目标企业很多值得深挖"或"这个方向匹配度低建议换方向"）
```

同时更新 /reports/scout/latest.md 为最新一份报告的内容。
